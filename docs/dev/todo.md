## 近期优先

### 1. 聚合搜索与聚合选集

当前流程还是先选一个片源，再看这个片源里的番剧和剧集。

后续更理想的方向是：

- 先聚合多个片源的搜索结果
- 尽量把同一部番剧归并
- 进入番剧后，再按“集”归类各个片源和各条线路
- 播放时优先选择可解析、可播放的线路

这会比“先选源、再一路走到底”更接近用户真实需求。

### 2. 线路页继续增强

当前线路页已经能展示：

- 剧集页地址
- 视频直链
- 格式
- 清晰度
- 解析错误

后续还可以继续补：

- 手动切换清晰度
- 标记推荐线路
- 记住上次成功线路
- 对明显失效线路做更明确的提示

### 3. 播放后的连续动作

现在播放成功后已经有播放页，也已经支持：

- 上一集
- 下一集
- 返回选集页
- 返回番剧页
- 返回首页
- 退出并保留外部播放器继续播放

后续优先补：

- 继续播放当前番剧

### 4. 测试继续补齐

当前自动化测试已经开始有了，但覆盖仍然偏少。

下一批值得优先加的测试：

- `internal/workflow` 里的播放分支
- `PotPlayer + m3u8` 是否会走本地代理
- 线路解析超时和失败状态
- 搜索结果排序和过滤逻辑

## 中期改进

### 1. TUI 状态路由继续收口

当前 TUI 已经能覆盖主要流程，但页面切换还是以多次启动页面程序为主。

后续可以考虑：

- 把 `goani tui` 收成一个长期运行的单一 TUI 程序
- 在内部切换 page/state，而不是频繁切页重建
- 进一步减少 Windows 终端下的闪烁

### 2. 搜索结果排序优化

当前搜索结果还缺少更稳定的本地相关性排序。

可以考虑加入：

- 完全匹配优先
- 前缀匹配优先
- 连续子串优先
- 更短、更像正片名的标题优先
- 降低特典、总集篇、剧场版等噪声结果权重

### 3. 播放器兼容层继续细化

`PotPlayer + m3u8` 的本地代理已经可用，但后续可能还需要继续增强：

- Cookie 转发
- 更多请求头透传
- 代理生命周期管理
- 代理播放状态可视化

### 4. 配置页继续完善

当前 TUI 配置页已经能编辑播放器和订阅源，但还可以继续增强：

- 配置项校验更直观
- 保存结果提示更细
- 对默认播放器和已配置播放器做更明显的视觉区分
- 增加更多配置项时保持页面结构不乱
