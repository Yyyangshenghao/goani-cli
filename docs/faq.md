# 常见问题

## `goani tui` 在普通终端里打不开怎么办？

`goani tui` 需要交互式终端支持。如果当前终端不支持 TUI，程序会提示你改用经典 CLI。

如果你只是想继续用终端命令，可以直接使用：

```bash
goani search 葬送的芙莉莲
goani play 葬送的芙莉莲
```

---

## 提示「未找到可用播放器」

先在 TUI 的 `配置` 页面里添加播放器路径，或者直接运行：

```bash
goani config player mpv "/path/to/mpv"
```

如果你已经填过路径，再确认一下路径是否真的指向可执行文件。

---

## `goani config player <name> <path>` 会自动设为默认播放器吗？

不会。这个命令只负责保存路径。

如果你要切换默认播放器，请单独运行：

```bash
goani config player default mpv
```

---

## 为什么我没设默认播放器，播放时它还是能播？

这是预期行为。

如果默认播放器为空，`goani` 会在你已经配置过路径的播放器里自动挑一个可用的，并在播放时把它写回默认播放器。这样可以减少第一次配置时的额外操作。

---

## PotPlayer 播放 `m3u8` 为什么有时会失败？

`m3u8` 是流媒体播放列表，不是单一视频文件。某些线路还会依赖请求头、防盗链或子 playlist 解析。

如果默认播放器是 `PotPlayer`，`goani` 可能会先启动本地 HLS 代理，再把本地地址交给播放器。这个过程是正常的兼容行为。

如果仍然失败，建议先试一下：

1. 换一条线路
2. 换成 `mpv`
3. 检查当前链接是否需要额外请求头

---

## 线路解析为什么要等一会儿？

当前线路页会并发解析视频直链和清晰度，但整体等待时间最多 5 秒。超时的线路会被标记为失败或超时，不会一直卡住页面。

如果某条线路总是超时，通常说明它本身就不太适合直接播放。

---

## 搜索结果太多或不太准怎么办？

可以先在片源结果里选一个更靠谱的源，然后在番剧列表里继续做本地过滤。

TUI 的番剧列表支持二次搜索，不会重新请求网络，适合逐步缩小范围。

---

## 如何打开或编辑 `config.json`？

最省事的方式是进入：

```bash
goani tui
```

然后打开 `配置`，再选择 `打开 config.json`。

你也可以直接手动编辑文件，位置通常是：

- Windows: `%USERPROFILE%\.goani\config.json`
- macOS/Linux: `~/.goani/config.json`

---

## `config.json` 和 `sources_cache.json` 有什么区别？

`config.json` 是主配置，保存播放器路径、默认播放器和订阅信息。

`sources_cache.json` 是缓存文件，保存程序运行时拉取到的媒体源数据，方便下次启动更快。

一般只需要关心 `config.json`，缓存文件交给程序管理即可。

---

## 搜索不到结果怎么办？

可以先检查这几件事：

1. 关键词是否太短
2. 网络是否正常
3. 当前片源是否有这部番
4. 是否需要换一个更准确的关键词

---

## 播放失败或视频无法加载怎么办？

可以按这个顺序排查：

1. 确认播放器路径已经配置
2. 换一条线路
3. 换另一个播放器
4. 如果是 `PotPlayer + m3u8`，优先尝试 `mpv`

---

## Windows 下提示 `goani 不是内部或外部命令` 怎么办？

1. 确认 `goani.exe` 所在目录已经加入 `PATH`
2. 重新打开终端
3. 再运行：

```powershell
goani version
```

---

## 怎么查看当前支持哪些媒体源？

运行：

```bash
goani source list
```
